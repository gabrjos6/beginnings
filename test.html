There's a better way to use HTML files: with an index.
An index file (index.html) is the index of the folder, or if the repository/main folder has no child folders, it's the index of the main repo/folder.
So when you name an HTML file index.html, the URL becomes gabrjos6.github.io/beginnings/ without the test.html.







Actually, it's all up to taste. I prefer this over beginnings/test.html, but you can change it back. Just rename index.html to test.html and delete this.
